Multiplication with a matrix¶

Multiplication by another matrix isn’t as important as the multiplication of a sparse matrix with a vector. Therefor, only two general methods are provided [2] [3]:

  • \(A * B\): A.matdot(B);
  • \(A^T * B\): A.matdot_transp(B);

with A being a sparse matrix (LLSparseMatrix) or a matrix proxy () and B being another sparse matrix or NumPy matrix.

These operations can also be optained throught the corresponding proxies (with the same efficiency):

  • A.matdot(B) is the same as A * B;
  • A.matvec_transp(B) is the same as A.T * B;

Views don’t implement matrix multiplication.

Multiplication with a NumPy matrix¶

Multiplication with a NumPy ndarray or matrix returns a dense NumPy ndarray.

A = LinearFillLLSparseMatrix(nrow=2, ncol=2)
B = np.mat('4 3; 2 1', dtype=np.float64) # NumPy matrix

print A
print B

C = A * B

print C
print type(C)

displays:

LLSparseMatrix [INT64_t, FLOAT64_t] of size=(2, 2) with 4 non zero values <Storage scheme: General and without zeros>
 1.000000   2.000000
 3.000000   4.000000


[[ 4.  3.]
 [ 2.  1.]]
[[  8.   5.]
 [ 20.  13.]]
<type 'numpy.ndarray'>

As you can see, despite the NumPy matrix being of type matrix, we return a NumPy ndarray [1].

This matrix multiplication only works when you multiply a sparse matrix by a NumPy matrix: multiplying a NumPy matrix by a CySparse sparse matrix is not allowed! Thus the muliplication B * A with the matrices above raises an exception.

Warning

Multiplication of a CySparse sparse matrix by a NumPy matrix is not allowed!

Footnotes

[1]This is general: CySparse only returns NumPy ndarrays.
[2]Some more mutliplication methods exist. In particular, the LLSparseMatrix class offers some other multiplication possibilities.
[3]Matrix by matrix multiplication is not as well supported as matrix by vector multiplication. This is work in progress.
